##ph

###Описание
программка для хранения в зашифрованном виде пар "ключ-значение". предоставляет консольный
интерфейс для добавления/удаления информации в шифрованный файл-базу данных, а 
также для просмотра и поиска по нему. в такой базе можно хранить пароли - всяко безопаснее, 
чем просто в текстовом файле. можно писать в неё короткие заметки, и никто, 
кроме вас не сможет их прочитать. можно оставлять записки другим пользователям компьютера,
типа как дома на холодильнике. есть поддержка нескольких пользователей.
короче, можно всё, на что хватит фантазии :)

###Дисклеймер
данная программа не претендует на звание надёжной криптосистемы для хранения важных конфиденциальных 
данных (по крайней мере, пока). как сказано выше, в текущем виде хранить в ней данные не намного более 
безопасно, чем в текстовом файле в открытом виде. просьба иметь это в виду, либо потом пенять 
исключительно на себя. больше того, **назначение программы - не столько скрыть** хранимые в базе данные 
от посторонних глаз, **сколько помочь** их **не забыть** их владельцу. 
и вообще, прога предоставляется как есть, без всяких гарантий, и далее по тексту, и вообще.

###Руководство пользователя
общий синтаксис команд таков:

`ph <команда> <опции>`

при запуске программы без параметров будет выведена краткая инструкция по использованию, 
её можно почитать. если же указана какая-нибудь команда, первым делом программа осведомится, 
кто вы есть-то вообще. каждый пользователь программы должен знать определённый passkey (пароль-ключ). 
по нему программа определяет, кто к ней обращается, то есть к какой (или, если угодно, чьей) группе 
данных выполнять запрос. пасскей - это одновременно и пароль, идентифицирующий пользователя, и ключ, 
которым шифруется запись (возможно, и не одна). логика обращения с пасскеем зависит от конкретного 
варианта использования. если вы, например, хотите хранить в базе секретные данные (пароли, 
явки или просто какие-нибудь тайные заметки или напоминалки), то ключ должен быть известен только вам. 
напротив, если вы хотите таким замысловатым образом обмениваться информацией с соседями по компьютеру, 
ключ должен бывть известен и им. разумеется, в одной и той же базе можно хранить и то, и другое, 
кол-во различных пасскеев в одной базе ничем не ограничено.

однако нужно быть осторожным. если, например, при добавлении записи в базу неверно ввести пасскей, 
программа молча добавит в базу запись и зашифрует её новым ключом, так как ей ничего не известно 
об уже существующих ключах, а даже если бы и было известно, это слишком ценная информация для 
потенциального взломщика, чтобы просто так её выдавать. именно поэтому пасскей при вводе не зашаривается 
звёздочками или точками, а вводится видимым текстом. нуу, может, ещё и потому, что я не знаю, как 
реализоватьтакое зашаривание. короче, бдите :)

вводить опции в командную строку необязательно. если не указать какие-то из них, программа 
переспросит. там, где есть значение по умолчанию, используется оно.

поддерживаются следущие команды:

**-add** - добавить запись в базу. опции:

*-key* - ключ, по которому потом можно выбрать значение из базы; это некое описание значения, 
хранимого  в базе. например, если запись содержит пароль, в ключе можно написать, от чего этот пароль.

*-val* - значение, хранимое в записи. то, ради чего всё и затеяно.

никто, конечно, не запрещает хранить пароли на месте их описаний, и наоборот. чтобы враг не догадался. 
тут всё ограничено, повторюсь, только вашей фантазией.

например, команда `ph -add -val 1234 -key "my pass"` сохранит в базе пару 'my pass' -> [1234].

не стоит забывать о том, что в linux, например, все команды, передаваемые командному интерпретатору через 
терминал, по умолчанию логируются в файл ~/.bash_history. если использовать указанную выше команду, 
в этот файл открытым текстом попадёт и ваш пароль. во избежание этого как раз и реализовано описанное 
выше поведение программы в случае, когда в командной строке указываются не все обязательные опции. 

**-list** - вывести все записи данного пользователя в базе. опции:
*-sym* - количество первых символов значений пар, выводимых в открытом виде. например, если пользователь 
хранит в базе пароли и не хочет, чтобы они выводились открытым текстом на экран, он может написать: 
`ph -list -sym 3`. по этой команде будут выведены все пары из базы с таким ключом, при этом будут 
отображены только первые три символа паролей. сесУриту, фигле :) по умолчанию открыто выводятся все 
символы.

**-get** - вывести все записи в базе с указанным ключом. опции:
*-sym* - см. выше.
*-key* - интересующий ключ. если его не указать, программа запросит его ввести.

**-rm** - удалить из базы записи с указанным ключом. опции те же, что и у **-get**.
допускается хранить несколько записей с одинаковым ключом. при удалении программа уточнит про каждую 
запись, нужно её удалять или нет.

сохранение изменений в базе осуществляется только при успешном выполнении всех действий, 
запрошенных оператором. рекомендуется иногда для проверки состояния базы 
выполнять команды *-list* и *-get*. мало ли что.
