##ph

###Описание
программка для хранения в зашифрованном виде пар "ключ-значение". предоставляет консольный
интерфейс для добавления/удаления информации в шифрованный файл-базу данных, а 
также для просмотра и поиска по нему. в такой базе можно хранить пароли - всяко без
опаснее, чем просто в текстовом файле. можно писать в неё короткие заметки, и никт
о, кроме вас не сможет их прочитать. можно оставлять записки другим пользователям компьютера,
типа как дома на холодильнике. есть поддержка нескольких пользователей.
короче, можно всё, на что хватит фантазии :)

###Дисклеймер
данная программа не претендует на звание надёжной криптосистемы для хранения важных конфиденциальных 
данных (по крайней мере, пока). как сказано выше, в текущем виде хранить в ней данные не намного более 
безопасно, чем в текстовом файле в открытом виде. просьба иметь это в виду, либо потом пенять 
исключительно на себя.
и вообще, прога предоставляется как есть, без всяких гарантий, и далее по тексту, и вообще.

###Руководство пользователя
общий синтаксис команд таков:

`ph <команда> <опции>`

опции необязательны. если не указать какие-то из них, программа переспросит. там, где есть 
значение по умолчанию, используется оно.

поддерживаются следущие команды:

**-add** - добавить запись в базу. опции:

*-key* - ключ, по которому потом можно выбрать значение из базы; это некое описание значения, 
хранимого  в базе. например, если запись содержит пароль, в ключе можно написать, от чего этот пароль.

*-val* - значение, хранимое в записи. то, ради чего всё и затеяно.

например, команда `ph -add -val 1234 -key "my pass"` сохранит в базе пару 'my pass' -> [1234]. 

**-list** - вывести все записи данного пользователя в базе. опции:
*-sym* - количество первых символов значений пар, выводимых в открытом виде. например, если пользователь 
хранит в базе пароли и не хочет, чтобы они выводились открытым текстом на экран, он может написать: 
`ph -list -sym 3`. по этой команде будут выведены все пары из базы с таким ключом, при этом будут 
отображены только первые три символа паролей. сесУриту, фигле :) по умолчанию открыто выводятся все 
символы.

**-get** - вывести все записи в базе с указанным ключом. опции:
*-sym* - см. выше.
*-key* - интересующий ключ. если его не указать, программа запросит его ввести.

**-rm** - удалить из базы записи с указанным ключом. опции те же, что и у **-get**.
допускаетсф хранить несколько записей с одинаковым ключом. при удалении программа уточнит про каждую 
запись, нужно её удалять или нет.

сохранение изменений в базе осуществляется только при успешном выполнении всех действий, 
запрошенных оператором. рекомендуется иногда для проверки проверять состояние базы 
командами *-list* и *-get*.
